Why Coding?

"I Thought This Was a Design Subject"

  • The practical project of this subject is to build an Experience Prototype: this needs to include integrated electronics (that's the "smart" bit)
  • To create the interaction effects you want using those electronic components, they need to talk to each other in the right way
  • It's much easier to control how they talk to each other by using a small computer as the core of the circuit (instead of doing it all in pure low-level electronics)
  • This computer is the microcontroller (i.e. Arduino), and we need to program it with code, to tell it how to control the electronics

But Also, More Generally:

  • Programming is a useful tool (a "transferable skill") for many domains: science, comms, games, design, art ...interesting and fun?
  • ...a highly structured, logical approach to solving problems, that you install in your brain
  • ...and then externalise to a computer
  • Software is made out of language: we can create tools just by describing them clearly enough


"Coding Literacy is the New Literacy"

What is Coding?

  • Writing step-by-step instructions in a way a computer can understand in order to solve a problem... kind of like a recipe
  • ...i.e., being so clear and specific about what you want done, that even a dumb literal computer can do it for you
  • ...using a programming language, which is halfway between:
    • vague-but-intuitive human languages, and
    • the precise-but-difficult language of mathematics
  • We'll be using the Processing and Arduino languages - very similar!
  • Programs are all a matter of input → processing → output
  • You may have already done programming without knowing it?
  • Warning! Get comfortable with "Error-Driven Development":
    • 30% - writing new lines of code
    • 70% - figuring out why they don't work (usually typos)

Why Processing?

  • You've already seen it
  • It's easy to install and get started with
  • It's a very graphical/multimedia-oriented language, created for designers & visual artists
  • It's very similar to Arduino language ...but it lets us start by focusing on just coding, without also troubleshooting electronic circuits (yet)
  • You might end up connecting your prototype electronics to it anyway

Data Types

"What kinds of information can I work with?"

  • Numbers:
    • int (integers/whole numbers): 1, 73542, -11, 0 ...
    • float (floating point numbers/decimals/fractions): 1.0, 0.5, 3.14159, -84632.1 ...
  • char (single characters): 'a', 'A', 'x', '&' ...
  • String (longer strings of characters): "Hello World", "Please enter password:"...
  • boolean: (true/false, yes/no answers): only true, false
  • A list of any of the above types ("array"): []
  • Lots of custom types i.e. PImage class for storing binary image data...

Variables

  • A variable is a container for storing data
  • It lets us give a clear, unique name to a piece of information
  • It has two parts:
    • a label (or "identifier") - make them as descriptive as possible!
    • the value being stored
  • To declare and initialize a variable: 
// Give me a new integer var called age and store 25 into it
int age = 25;

// Once created, you can change what's stored in it
age = 30;

// You CAN'T change the TYPE of data you store in it
age = "Thirty";   // ❌ NO

String ageInWords = "Thirty";    // ✅  OK

Functions

  • A function is a named container for grouping related lines of code
  • One of the most important building blocks of code: most programs are just functions calling each other (divide and conquer)
  • They can sometimes have inputs (arguments) and outputs (return values) - in this way they're like miniature programs
  • There are two aspects to a function:
    • defining the function (unless it's pre-defined)
    • using (a.k.a. running/calling/executing/invoking) the function
  • Processing comes with a lot of pre-defined functions ready to use. These are like handy little reusable pieces of code, which you can run to carry out common tasks.

Functions II

  • Most of the time we'll run pre-defined or "built-in" functions that Processing makes available to us: 
    // Run the fullScreen function
fullScreen();

// Run the rect function and give it some info about what to draw
rect( 0, 0, 20, 20 );

// Run the random function and save the number it returns into a variable
int ledBrightness = random( 0, 255 );
  • Sometimes we'll define a function which Processing expects to exist, and it will automatically be run for us at the right time: 
    void draw(){

  println( frameCount );

} // Need curly brackets at start and end of definition!
  • You might even define and later run your own custom functions?

Conditionals

  • Conditionals, a.k.a branching, a.k.a if-else statements, let our programs do different things at different times, depending on the answers to questions we ask about our data: 
    
if( buttonHeldTime > 1000 ){
  openDoor();
} else {
  println("Still closed...");
}
  • The expression inside the parentheses buttonHeldTime > 1000 is the condition, the question we are asking; it always has a yes/no answer, i.e. a boolean true/false value.
  • You can ask lots of questions to compare data in different ways: ==, <, >, <=, >=, !=
